## Bugs / Code quality

### Fixed 🔴

- **`components/HiddenCostsCalculator.tsx:41` — `useMemo` called conditionally
  via short-circuit (`citySlug === "paris" || !!useMemo(...)`).** Violates
  React rules-of-hooks. Refactored to call the hook unconditionally first,
  then OR with the short-circuit. ESLint errors `react-hooks/rules-of-hooks`
  and `react-hooks/preserve-manual-memoization` both cleared.

- **`components/GlobalSearch.tsx:37` — setState inside effect on mount to
  hydrate `?q=` from URL.** Matches the codebase convention

  `0gmb8zs~7hdq_.js` (2.32 MB) and `0b0hoxziznvmd.js` (2.32 MB — identical

  certainly the recharts bundle pulled in by `PurchaseSimulator`,
  `CompatibilityQuiz`, `OwnerScoresCard`, `Climate2040Card`, etc. Worth a
  future spike to dynamic-import recharts behind a `dynamic(..., { ssr: false })`
  wrapper per consumer — would shave ~600 kB from First Load on city pages.
  Not auto-fixable.

### Open 🟢

- **Many `recharts` SSR warnings during build:** `The width(-1) and height(-1)
  of chart should be greater than 0`. Recharts' `ResponsiveContainer` reads
  parent box during SSR; the parent measures to `-1` until hydration. Cosmetic
  build noise, but should also be addressed by the dynamic-import change above
  so recharts never runs during the static build.
